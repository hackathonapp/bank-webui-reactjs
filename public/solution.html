Branch & ATM Use Case Solution About

The registration communicates to the "Bank" Microservices - where inputs are verified, validated and collected. It follows an orchestrated workflow, different API invocation, thus the 3-step registration.

Customer Data Input Complete all the required fields and submit the customer information form SMS OTP Validate transaction using the One-Time-Pin sent to mobile number Know Your Customer Upload ID image file. and sign the specimen card embed in the form

The microservice utilizes other solutions provider in sending notification, log aggregation, and object storage. Texts from the uploaded ID image file are extracted using IBM Developer's MAX OCR Model

Hosted on Red Hat Openshift
